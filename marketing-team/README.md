# Marketing Team

Weekly LinkedIn content pipeline for Caiman — Senior PM @ Amazon + coach de mentoria para PMs.

5 sub-agents, run sequentially every Monday. Outputs human-reviewed before publishing.

## Structure

```
marketing-team/
  CLAUDE.md                  ← governance + hard rules (read this first)
  lead-agent-prompt.md       ← paste this into Claude Code to run the pipeline
  README.md                  ← you are here

  brand/                     ← stable, edit rarely
    brand-dna.md             ← positioning + differentiators + anti-positioning
    brand-voice.md           ← 7 voice rules + banned phrases
    icp-cards.md             ← 5 ICPs with hook angles + conversion paths
    brief-template.md        ← structure for Agent 2 output
    hooks-history.md         ← winners, losers, 10-framework catalog

  inputs/                    ← refresh weekly before pipeline run
    granola-context.md       ← recurring mentee patterns (from SOP-01)
    post-performance.csv     ← LinkedIn analytics export
    competitor-content/      ← screenshots/links per competitor
    reviews/                 ← mentee feedback (consent-gated)

  outputs/                   ← generated by agents, overwritten each cycle
    competitive-brief.md     ← Agent 1
    creative-brief.md        ← Agent 2
    hooks.md                 ← Agent 3
    content-variations.md    ← Agent 4
    performance-report.md    ← Agent 5
```

## Cadence

| Day | What |
|---|---|
| **Sunday PM** | Refresh `inputs/`: export LinkedIn analytics → `post-performance.csv`, run SOP-01 → `granola-context.md`, dump new competitor screenshots into `competitor-content/<name>/` |
| **Monday AM** | Open Claude Code in `marketing-team/`. Paste `lead-agent-prompt.md`. Pipeline runs end-to-end (~20-40 min). |
| **Monday PM** | Human review of `outputs/`. Edit. Schedule 3-5 posts for the week. |
| **Friday** | Read `outputs/performance-report.md`. Note "Research Focus for Next Week" — this guides Sunday's competitor scrape. |

## First-run setup (one-time)

Before running the pipeline for the first time:

1. **Fill the TBDs in `brand/hooks-history.md`** — actual hook text + frameworks for the 1 winner (Nubank post) and 5 losers. Without this, Agent 3 has no real anchor.
2. **Export LinkedIn analytics** → populate `inputs/post-performance.csv` with the last 30-60 days of posts (one row per post).
3. **Run SOP-01 on last 30 days of Granola notes** → paste structured output into `inputs/granola-context.md`.
4. **Seed `inputs/competitor-content/`** with 2-3 folders (igor-dias/, lipe-boff/, etc.) and 3-5 screenshots each.
5. **Validate `brand/brand-dna.md` and `brand/icp-cards.md`** — these were drafted from inference. Fix anything wrong before the pipeline anchors on them.

## How to run

From inside `marketing-team/`, in a fresh Claude Code session:

```
[paste contents of lead-agent-prompt.md]
```

Lead Agent will dispatch sub-agents in order and stop at any validation failure.

## Hard rules (also in CLAUDE.md)

1. No fabricated data. Mark `[unknown]` or `TBD` if not in inputs.
2. PT-BR by default.
3. No emoji in body copy. Banned phrases live in `brand/brand-voice.md`.
4. One decision per brief — Brief Writer commits to ONE angle.
5. Real mentee patterns only (from `inputs/`), anonymized.
6. No auto-publish. Human reviews every post.

## Not in scope (intentionally)

- Email, ads, podcast, newsletter — single channel by design until LinkedIn pipeline is dialed
- Auto-scheduling/publishing — human in the loop is the point
- Multi-language (EN/PT) parallel — PT-BR primary; EN only on demand
